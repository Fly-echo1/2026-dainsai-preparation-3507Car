# 自动驾驶小车 — 软件设计规格

> 2024 全国大学生电子设计竞赛 H 题 · MSPM0G3507 平台
> 
> 设计日期：2026-07-12

---

## 一、题目要求

### 场地

白底黑线，矩形 + 两端半圆弧 "田径跑道" 形状。A、B、C、D 四个站点位于四角。

### 四问

| 问号 | 路径 | 限时 |
|------|------|------|
| Task 1 | A→B 直线停车 | ≤15s |
| Task 2 | A→B→C→D→A 完整一圈 | ≤30s |
| Task 3 | A→C→B→D→A 交叉路径一圈 | ≤40s |
| Task 4 | 交叉路径 (A→C→B→D→A) ×4 圈 | 越短越好 |

### 约束

- 轮式小车，不能后退，不能装摄像头
- TI MSPM0 系列 MCU
- 车头 8 路灰度传感器（P1 最右 → P8 最左）
- 行驶场地无 A/B/C/D 字符标记

---

## 二、整体架构

```
┌─────────────────────────────────────────┐
│              main.c (应用层)              │
│  按键选任务 → 倒计时 → Task_Run()        │
├─────────────────────────────────────────┤
│           Control.c (控制层)              │
│  StraightDrive / ArcFollow / TurnToAngle │
│  IsStation / IsOffTrack / StationAlert   │
├─────────────────────────────────────────┤
│              驱动层                       │
│  灰度 | 陀螺仪 | 编码器 | 电机 | OLED/蓝牙│
└─────────────────────────────────────────┘
```

- **架构风格**：阻塞式状态机（方案 A），每个路段封装为阻塞函数，直到到站/超时才返回
- **设计依据**：基于 MSPM0 参考代码（`2024-Electronic-Design-Competition-master`）的解题逻辑，从头重设计

---

## 三、状态机 — flag 管理

```
flag = 0: 停车/等待
flag = 1: AB 直线段
flag = 2: BC 弧线段
flag = 3: CD 直线段 / BD 直线段
flag = 4: DA 弧线段
```

flag 在 `Control_StraightDrive()` 和 `Control_ArcFollow()` 内部由站点检测触发切换，每个 flag 切换时触发声光提示。

---

## 四、控制层三大核心函数

### 4.1 直线段 — `Control_StraightDrive(target_yaw, distance_m)`

| 环节 | 策略 |
|------|------|
| **方向控制** | 陀螺仪 Yaw 角增量式 PID → `steer_out`，保持航向 |
| **速度控制** | 编码器增量式 PID → `speed_out`，保持基础速度 |
| **合成输出** | `left = base + speed_out + steer_out`，`right = base + speed_out - steer_out` |
| **退出条件** | 站点检测触发 或 里程达标 |
| **脱线检测** | 不检测（直线段仅靠站点判据退出） |

### 4.2 弧线段 — `Control_ArcFollow(distance_threshold_m)`

| 环节 | 策略 |
|------|------|
| **巡线方式** | 灰度查表法：8 路传感器 → 找到第一个黑线 → 查表得目标角速度 |
| **角速度闭环** | 陀螺仪 gz 角速度增量式 PID → 差速修正量 |
| **合成输出** | `left = ARC_SPEED + omega_out`，`right = ARC_SPEED - omega_out` |
| **退出条件** | 站点检测触发 或 里程达标 |
| **脱线检测** | 8 路全白持续 N 次 → 减速继续（不退出），作为保护机制 |

**灰度查表**（传感器从左 P8 → 右 P1）：

| 传感器 | P8 | P7 | P6 | P5 | P4 | P3 | P2 | P1 |
|--------|-----|-----|-----|----|----|----|-----|-----|
| 位置 | 最左 | 左 | 左中 | 中左 | 中右 | 右中 | 右 | 最右 |
| 目标角速度(°/s) | -20 | -10 | -5 | -3 | +3 | +5 | +10 | +20 |

不同路段可微调此表（如 DA 弧线加大外侧权重）。

### 4.3 原地转弯 — `Control_TurnToAngle(target_yaw)`

- 固定转速差速转弯（一正一反）
- 陀螺仪 Yaw 判断到达目标角度（误差 < 3°）

## 五、站点检测 — 差异化判据

不同任务、不同站点采用不同判据，基于传感器物理位置与路径方向的关系。

### Task 1

| 站点 | 条件 | 去抖 |
|------|------|------|
| B 点 | 任一路灰度检测到黑线 | 30 |

### Task 2

| 站点 | 条件 | 去抖 |
|------|------|------|
| B 点 | 任一路灰度检测到黑线 | 30 |
| C 点 | 全部 8 路白线 | 5000 |
| D 点 | 右侧 4 路 (P1-P4) 检测到黑线 | 30 |
| A 点 | 全部 8 路白线 | 7000 |

### Task 3 / 4

| 站点 | 条件 | 去抖 |
|------|------|------|
| C 点 | 左侧 4 路 (P5-P8) 检测到黑线 | 50 |
| B 点 | 全部 8 路白线 | 7000 |
| D 点 | 右侧 4 路 (P1-P4) 检测到黑线 | 50 |
| A 点 | 全部 8 路白线 | 7000 |

## 六、脱线检测

- **仅弧线段**生效。8 路全白 → 脱线。
- 脱线时**不退出函数**，减速继续（降为基础速度的 50%），等待灰度重新检测到黑线。
- **直线段不检测脱线**。

---

## 七、各 Task 执行流程

### Task 1：A→B 直线停车

```
Control_StraightDrive(0°, 1.5m)
  → B 站到站 → 停车 → 声光提示
```

### Task 2：A→B→C→D→A 完整一圈

```
flag=1: Control_StraightDrive(0°, 1.5m)     → B 站声光
flag=2: Control_ArcFollow(ARC_BC_DIST)       → C 站声光（全白）
flag=3: Control_StraightDrive(180°, 1.5m)    → D 站声光（右侧黑）
flag=4: Control_ArcFollow(ARC_DA_DIST_T2)    → A 站声光（全白）→ 停车
```

### Task 3：A→C→B→D→A 交叉路径一圈

```
flag=1: Control_StraightDrive(0°, ?m)        → C 站声光（左侧黑）
        Control_TurnToAngle(90°)             → 原地右转 90° 进入 CB 弧线
flag=2: Control_ArcFollow(ARC_CB_DIST)       → B 站声光（全白）
flag=3: Control_StraightDrive(90°, 1.5m)     → D 站声光（右侧黑）
flag=4: Control_ArcFollow(ARC_DA_DIST_T3)    → A 站声光（全白）→ 停车
```

### Task 4：交叉路径 ×4 圈

```
for lap = 1..4:
    Task 3 流程
    每圈结束后 Control_TurnToAngle(180°) 掉头（非最后一圈）
```

## 八、PID 策略

三套增量式 PID，全部带输出限幅：

| PID | 输入 | 目标 | 输出 | 范围 |
|-----|------|------|------|------|
| `pid_angle` | 陀螺仪 Yaw | 目标航向 | `steer_out` | [-40, 40] |
| `pid_speed` | 编码器速度 | 基础速度 | `speed_out` | [-30, 30] |
| `pid_omega` | 陀螺仪 gz (°/s) | 查表角速度 | `omega_out` | [-20, 20] |

公式：增量式 `Δu = Kp * (e - e?) + Ki * e + Kd * (e - 2e? + e?)`

角度 PID 需处理 Yaw 角在 ±180° 边界的环绕问题。

## 十、调试监控

- **OLED**：实时显示任务号、当前路段、里程、速度、灰度状态
- **蓝牙**：JSON 日志输出到电脑，格式 `{"event":"seg_start","seg":"AB","yaw":...}`
